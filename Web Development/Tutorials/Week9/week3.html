Module Expectations

My 1st expectation is to master the fundamentals of semantic HTML so that I can structure web pages in a way that is both logical for developers and accessible for screen readers.

For my 2nd expectation, I want to learn how to connect CSS files to my HTML documents effectively, allowing me to transform basic text into visually engaging layouts with custom fonts and colors.

My 3rd expectation is to finish this module with a complete, portfolio-ready project that demonstrates my ability to troubleshoot code and solve common design problems independently.

Today, I learned how to use specific formatting tags like superscript to handle ordinal numbers, and I reinforced my understanding of block-level elements like headings and paragraphs.

Reflection

One of the most challenging aspects of learning to code is remembering to close every tag while trying to articulate ideas. When I take a moment to reflect on this difficulty, I realize it stems from the need to switch back and forth between logical coding syntax and creative expression.

I found it surprisingly interesting to reflect on how the rigid structure of HTML actually helps clarify my thinking. By strictly using paragraph tags, I was forced to group my ideas into distinct units rather than rambling.

As I continue to reflect on my progress, I feel increasingly confident in my ability to create semantic markup. I now understand how to nest elements correctly and how to use headings to create a proper hierarchy.

As I look back on this project, I have had to reflect deeply on the progress I have made in understanding web structure. At the start, I found the complexity of tables and lists to be quite overwhelming. Specifically, keeping track of table rows and data cells while trying to apply borders and padding required a lot of trial and error. It was challenging to ensure that the table looked professional rather than just a cluttered mess of data.

What I found most interesting and surprising was how much control CSS gives you over simple elements. I never realized that a few lines of code could change a standard bullet point into a roman numeral or a square. This level of customization makes the user interface feel much more intentional and polished. It was also surprising how helpful IDs are for targeting very specific parts of a page, like a footer or a contact link.

Here are some specific things I found challenging during this process:

  • Managing the syntax for nested table elements without errors.
  • Understanding the difference between class selectors and ID selectors.
  • Ensuring that the color contrast remained readable while styling.

Despite those challenges, I am now feeling very confident about several areas:

  1. Writing clean, semantic HTML for headings and paragraphs.
  2. Applying CSS hover effects to make tables interactive and engaging.
  3. Using various list-style-type properties to organize information clearly.

Overall, this exercise has taught me that web development is about both logic and aesthetics. By combining structural HTML with creative CSS, I can build pages that are both functional and beautiful. I look forward to applying these skills to more complex layouts in the future.
