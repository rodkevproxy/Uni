Action Impact Simulator

Technical Description

The page implements an interactive "Action Impact Simulator" using a dynamic combination of HTML, CSS, and JavaScript. The layout relies on semantic HTML5 tags and utilizes CSS Grid (display: grid with auto-fit and minmax) to create a fully responsive gallery of actionable cards that seamlessly adapts to different screen sizes. Visual feedback is enhanced through CSS transitions (such as transform: translateY and box-shadow) applied dynamically via an active class. The core interactivity is driven by custom JavaScript functions that manage a running score state (totalScore). When a card is clicked, the script toggles a .selected CSS class, calculates the new score, and immediately manipulates the Document Object Model (DOM) to update the feedback text and dynamically swap the page's background image based on predefined point thresholds.

Accessibility

Accessibility is actively integrated into both the markup and the scripting of this interactive tool. Because the interactive cards are built using <div> elements rather than native buttons, they were given a tabindex="0" attribute to make them focusable via keyboard navigation. A custom CSS :focus state (outline: 3px solid var(--accent-yellow)) ensures a highly visible indicator appears when a card is tabbed onto. Crucially, a dedicated JavaScript event listener was implemented to detect the "Enter" keypress, programmatically triggering the click event so keyboard-only users can interact with the simulator effectively. Additionally, all images feature descriptive alt attributes, and semantic landmarks assist screen readers in parsing the page content.

Splash Screen

Technical Description

The page is constructed using a semantic HTML5 hierarchy, utilizing a <header> for site identity, a <main> container for the primary narrative, and <section> elements with unique IDs to define clear thematic boundaries. The layout leverages CSS for a clean, professional aesthetic, employing custom properties for consistent branding and a responsive structure that ensures content remains readable across devices. Key styling choices include the use of border-radius and box-shadow to give images a modern feel, while the Table of Contents uses a distinct background and border-left to improve scannability. Interactivity is integrated via internal anchor links and a "Back to Top" button implemented with position: fixed and a high z-index, allowing users to navigate long-form content effortlessly without heavy JavaScript dependencies.

Accessibility

Accessibility was addressed by ensuring high navigational clarity and adhering to visual standards. I used semantic tags like <nav> and <ul> to allow screen readers to interpret the site structure accurately and provided descriptive alt text for all images to convey the meaning of the data graphics to visually impaired users. The inclusion of an internal Table of Contents and a persistent "Back to Top" link assists users with motor impairments or those using keyboard navigation to jump between sections quickly. Furthermore, the CSS defines clear hover states and underlines for links to ensure they are easily distinguishable from regular text.

Challenges and Lessons Learned

One significant challenge was interpreting the complex coursework requirements and translating them into technical solutions. For example, the brief required not just a static website, but an interactive experience that adhered to strict accessibility and validation standards. Figuring out exactly how to implement features like the semantic HTML structure for the Content page or the specific timed-redirect requirements for the Splash screen meant I had to spend extra time deconstructing the assignment into actionable coding steps before I could even begin typing.

Regarding teamwork, I learned that strong communication and establishing a shared foundation like an agreed upon CSS stylesheet and semantic HTML structure early in the project is vital. It saves hours of debugging and refactoring when merging files later on. Ultimately, these challenges improved not just my technical coding abilities, but my overall approach to project management and collaborative problem-solving.
